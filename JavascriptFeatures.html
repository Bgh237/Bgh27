My Eureka Moment

The problem I was having was that I was over complicating things in my head and thinking things were much harder than they acually were. I used a number of sources on the internet to try and figure out the issues I was having with Arrays but all I seemed to come across was solutions for more difficult and adanced options. I kept working at my problems and then everything seemed to click in to place and I started to look at things in a more simplistic way. Below I have listed a number of features of Javascript that you are likely to come across whilst learning. Variables

Variables are an important part to learn in JavaScript. Variables allow you to assign string or integers to a word. They are basically a named memory location. This becomes very handy when using very large numbers or calculations. Instead of typing out the whole number or calculation, you can just type the name of the variable and JavaScript will know you want to use whatever has been assigned to that variable. This will speed up your code writing a great deal and makes your code a lot easier to read for other developers and web browsers. Below is an example of variables in use:

Finally, the variable z is created and the variable x is added to the variable y. As x is 5 and y is 6, z becomes 5 + 6 which equals 11. If you were to print z it would print the number 11.

This was something that I realised early on was important as it made it much easier for me to understand how id's work.

Loops

Loops are an important feature within JavaScript. There are two main types of loops, a 'for' loop and a 'while' loop. Both of these loops will run a code when certain criteria is met. If the input does not meet the criteria, the code will not run. Below is an example of a 'for' loop and a 'while' loop:

While Loop:

From the example of the 'while' loop we can see that the variable i is defined outside of the loop as well as the variable text, which has been left as an empty string. i has been defined as 0 so when we start the loop we know that i is 0. The loop specifies that while i is less than 10 this code will run. As we know, i is 0 to start with so we enter the loop. The loop then tells us that we need to add the string to the text variable. Notice the += sign here. This is very important to remember as this allows you to add something on to an existing variable. Remembering this will help a lot when you are asked to code more complicated things. After the string is added we also add the value of i to the string, which is this case is 0. Adding something to a string like this is known as concatenation. The text is then saved to the variable for future use in the loop. We then see i++ in the code. This is a simpler way to say i + 1 and turns i from 0 to 1. We then go back to the while condition and check that i still meets the condition, which it does, so we run the code again. This will continue until i no longer meets the condition outlined by the loop. Once this heppens we then print the combined text to the HTML page and out output can be seen in the picture on the right.

For Loop:

As you can see from the example of the 'for' loop, it is very similar to the 'while' loop. The major difference is that the variable i is defined within the parenthesis of the 'for' loop rather than outside it like with the 'while' loop. You will notice that the i++ is also within the parenthesis but everything else is very much the same. The text variable is defined outside the loop and added to within the loop like before. The 'for' loop makes your code look much cleaner and then things get more complicated it is generally easier to understand than a while loop. However, there are some confusing aspects to the 'for' loop that confused me at first. As we have mentioned there are 3 parts to the 'for' loop, the allocation of a number to the variable, the condition and the addition/subtraction of i. Although these are all in the parenthesis, the loop does not action all of these before entering the loop. First it will take the assigned value of the variable and check it against the condition. If the variable meets the condition specifications then it will enter the loop. Once it enters the loop it will action the code within the loop. It will then go to the addition/subtraction and in this case add 1 to the variable i. It will then check this against the condition and enter the loop again. It will continue to do this until the variable i no longer meeters the specifications of the condition and once that heppens it will leave the loop and print the text variable to the HTML page as with the 'while' loop.

Loops are something that I defintely struggled with but I think my major problem with it was over complicating it in my mind. It is important to remember that the variable i is used only to specify how many times the loop will run and has no impact on anything else outside of the loop. One piece of advice I would give to you is that if you are struggling with loops, try to uncomplicate your thoughts and keep it simple. This helped me a lot when first learning loops.

Arrays

Arrays are very useful within JavaScript. Arrays allow you to store multiple values in a single variable. This may not sound that useful but if, for example, you had 100 students and you wanted to print all of their grades to the console, without an array you would have to type out each grade and assign each grade to its own variable. This would be very time consuming and clog up your page. Arrays allow you to create one variable and recall any value within the array by typing the name of the array and the position of the value within the array. The position needs to be within square brackets, for example, array[1]. It is very important to know that within an array the first value will be at position 0 and not 1. This caused me a little bit of confusion at first but the more you practice with arrays this will become second nature to you. New values can be added to an array at any time and values can also be replaced within an array so once you have created an array they are very flexible. Below is an example of an Array within a loop:

As you can see from the example, an array has been created for different types of fruit. the fruits.length feature returns the length of the array so in this case it would return 4 as there are 4 values in the array. This can cause some confusion as we said earlier that the array starts from position 0, however when it comes to length we simply count how many values are in the array and ignore any postions. When we enter the loop we have specified that i is 0 and that the amount of times we go through the loop will be less than the length of the array. We now enter the loop and add to the text variable as we have met the conditions of the loop. You will see that fruits[i] is added to the text variable. This is taking the value of the fruits array that matches the postion i, which in this case would be 0 so we take the value that is in position 0 of the array. This is then added to the text and we continue with the loop. As we have added 1 to the variable i that now become 1, which meets the condition so we go in to the loop and now we added the value that is in the furits array at position 1 as i is now 1. This continues until the condition is no longer met and we come out of the loop and print the text variable to the HTML page. Please note that when using an array within a loop it is important to set the condition of the loop to less than the length of the array. This is useful becomes often things will be added to the array so the size will grow. If we set the condition to a specific figure, the array could quickly outgrow that number and our loop is no longer effective. It is also important to set it less than the length due to the postions of the values in the array. As mentioned earlier if an array has 4 values its positions will be 0, 1, 2, 3 so i can only go up to 3 to bring back a value. If we set the condtion to less than or equal the length we would get i is 4 and as there is nothing in position 4 this will return nothing. This is something that I had to get my head around as it seems quite contradictory but the more you do it the more it sticks with you.
